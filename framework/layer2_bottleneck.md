# Layer 2: Bottleneck Identification (瓶颈识别)

## English

### Objective
Find the **narrowest link** in the supply chain where supply cannot expand quickly enough to meet demand.

### Supply Chain Map

```
AI Application → Cloud Computing → GPU/ASIC → HBM → DRAM → Wafer → Equipment → Materials
```

### Bottleneck Identification Method

1. **Draw the supply chain map** (from downstream to upstream)
2. **Find the narrowest link** (where capacity expansion is hardest)
3. **Calculate supply-demand gap**

### Examples of Bottlenecks (2025-2026)

| Bottleneck | Why Narrow? | Winners |
|------------|-------------|---------|
| **HBM** (High Bandwidth Memory) | 2025-2026 global capacity booked by NVIDIA | $MU, $SKHynix |
| **EML Lasers** | Only 4 global suppliers | Coherent, Lumentum, OE Solutions, Mitsubishi |
| **CW Lasers** | Critical for CPO solutions | $POET, $SIVE |
| **Advanced Packaging** | CoWoS capacity constrained | $AMAT, $LRCX |

### Supply-Demand Gap Calculation

```
Gap = 2026 Demand - 2025 Capacity × (1 + Expansion Rate)

If Gap > 0 → Pricing power shifts to suppliers
If Gap < 0 → Oversupply, avoid this segment
```

### Serenity's Words

> "The best trades are where supply can't expand for 18+ months, but demand is accelerating. That's when pricing power shifts to suppliers."

### Analysis Steps

1. **Identify key components** in the supply chain
2. **Research capacity expansion plans** (company announcements, earnings calls)
3. **Calculate time to expand** (wafer fabs: 18-24 months, equipment: 12-18 months)
4. **Quantify the gap** (demand vs. supply)
5. **Identify pricing power** (can suppliers raise prices?)

## 中文

### 目标
找到供应链中**最窄的环节**，即供应无法快速扩张以满足需求的环节。

### 供应链地图

```
AI 应用 → 云计算 → GPU/ASIC → HBM → DRAM → 晶圆 → 设备 → 材料
```

### 瓶颈识别方法

1. **画出供应链地图**（从下游到上游）
2. **找到最窄的环节**（产能扩张最难的）
3. **计算供需缺口**

### 瓶颈示例（2025-2026）

| 瓶颈 | 为什么窄？ | 赢家 |
|------|-----------|------|
| **HBM**（高带宽内存） | 2025-2026 全球产能被 NVIDIA 包圆 | $MU, $SKHynix |
| **EML 激光器** | 全球只有 4 家供应商 | Coherent, Lumentum, OE Solutions, 三菱 |
| **CW 激光器** | CPO 方案的关键 | $POET, $SIVE |
| **先进封装** | CoWoS 产能受限 | $AMAT, $LRCX |

### 供需缺口计算

```
缺口 = 2026 年需求 - 2025 年产能 × (1 + 扩产率)

如果缺口 > 0 → 定价权转移到供应商
如果缺口 < 0 → 供过于求，避免这个环节
```

### Serenity 的原话

> "最好的交易是：供应 18+ 个月无法扩张，但需求在加速。这时定价权转移到供应商。"

### 分析步骤

1. **识别供应链中的关键组件**
2. **研究产能扩张计划**（公司公告、财报电话会议）
3. **计算扩张时间**（晶圆厂：18-24 个月，设备：12-18 个月）
4. **量化缺口**（需求 vs. 供应）
5. **识别定价权**（供应商能否提价？）
